# Specification: PRD-LEDG-004 - Interlocking Cryptography

**Traceability Links:** - [Enhancement] PRD-LEDG-004

## 1. Domain Entities (Deltas)

### 1.1 `JournalEntry` (Appended Fields)
* `entry_hash: String`: A 64-character SHA-256 hexadecimal string.

## 2. Cryptographic Formulas

### 2.1 The `JournalEntry` Hash (The Cause)
1. The hashing algorithm MUST be `SHA-256`.
2. The input payload MUST be a concatenated UTF-8 string:
   `{id}{correlation_id}{created_at_unix_timestamp}{postings_data}`
3. The `postings_data` MUST be generated by iterating over all associated `Posting` entities in a deterministic order (e.g., sorted by account ID) and concatenating: `{account_id}{amount}{direction}` for each posting.

### 2.2 The `LedgerState` Hash (The Effect)
1. The state hashing formula defined in PRD-LEDG-002 and PRD-LEDG-003 MUST be appended with the `entry_hash`.
2. **Updated Formula:** `SHA-256(previous_state_hash + current_balance + version + updated_at_unix_timestamp + entry_hash)`

## 3. Execution Rules
1. `JournalEntry::new()` MUST calculate and assign the `entry_hash` immediately upon instantiation.
2. `LedgerState::apply_journal_entry()` MUST extract the `entry_hash` from the provided `JournalEntry` reference and inject it into the state hash calculation.
